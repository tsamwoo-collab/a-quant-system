# 飞书多维表格配置指南

## 📊 功能说明

每日交易信号将自动写入飞书多维表格，方便历史查看和统计分析。

**数据表结构**:
| 字段 | 说明 |
|------|------|
| 日期 | 交易日期（主键） |
| 市场状态 | ADX趋势状态 |
| 买入信号 | JSON格式信号列表 |
| 卖出信号 | JSON格式信号列表 |
| 当前持仓 | JSON格式持仓列表 |
| 今日平仓 | JSON格式平仓列表 |

---

## 🔧 配置步骤

### 第一步：创建飞书应用

1. **访问飞书开放平台**
   - 打开 https://open.feishu.cn/app
   - 登录你的飞书账号

2. **创建企业自建应用**
   - 点击"创建企业自建应用"
   - 应用名称: "量化信号记录"
   - 应用描述: "自动记录每日交易信号"
   - 点击"创建"

3. **获取凭证**
   - 进入应用详情页
   - 在左侧菜单选择"凭证与基础信息"
   - 复制保存:
     - **App ID**: `cli_xxxxxxxxxxxxx`
     - **App Secret**: `xxxxxxxxxxxxxxxxxxxx`

### 第二步：配置权限

1. **开通权限**
   - 在应用详情页，选择"权限管理"
   - 搜索并开通以下权限:
     - `bitable:app` - 查看、评论、创建和编辑多维表格
     - `bitable:app:readonly` - 以只读方式获取多维表格信息

2. **申请发布**
   - 在"版本管理与发布"
   - 点击"申请发布" → "创建版本"
   - 版本号: 1.0.0
   - 点击"申请发布"并等待审核通过（通常几分钟）

### 第三步：创建多维表格

1. **在飞书中创建表格**
   - 打开飞书客户端或网页版
   - 点击"+" → "多维表格"
   - 表格名称: "量化信号记录"
   - 点击"创建"

2. **获取 app_token**
   - 打开创建的多维表格
   - 查看 URL，格式类似:
     ```
     https://xxx.feishu.cn/base/xxxxxxabcdefgh
     ```
   - **app_token** 就是 `xxxxxx` 部分（base/后面的字符串）

### 第四步：填写配置文件

编辑 `config/feishu_bitable.json`:

```json
{
  "app_id": "cli_xxxxxxxxxxxxx",
  "app_secret": "xxxxxxxxxxxxxxxxxxxx",
  "app_token": "xxxxxx",
  "table_id": ""
}
```

**注意**: `table_id` 可以留空，系统会自动创建数据表。

## 🚀 自动运行

配置完成后，每日信号脚本会自动：
1. 生成交易信号
2. 推送到飞书群
3. **写入多维表格**（新增）

无需手动操作，每天15:30自动执行。

## 📖 查看数据

1. **打开多维表格**
   - 在飞书中打开"量化信号记录"表格
   - 查看"每日信号"数据表

2. **数据展示**
   - 系统会自动创建"每日信号"数据表
   - 包含完整的历史信号记录
   - 支持飞书多维表格的所有功能（筛选、排序、图表等）

## ⚠️ 常见问题

### Q1: 提示"获取 token 失败"?
- 检查 App ID 和 App Secret 是否正确
- 确认应用已发布并审核通过
- 检查网络连接

### Q2: 提示"权限不足"?
- 确认已开通 `bitable:app` 权限
- 确认应用已发布
- 重新获取 tenant_access_token

### Q3: table_id 填什么?
- 可以留空，系统会自动创建数据表
- 如果要使用已有数据表，需要手动填入 table_id
- 获取方式: 飞书多维表格 → 右键数据表 → 查看详情 → 复制 table_id

### Q4: JSON字段如何查看?
- 买入信号、卖出信号等字段存储为 JSON 字符串
- 可以使用飞书的公式字段进行解析
- 或导出后用其他工具分析
